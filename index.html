Home Lab Setup Details

This project started with a simple, practical goal: to solve the single-device limitation of my primary internet source, wifi.id@home. The initial setup was just a basic router to share this connection. As I progressed through my CCNA studies, I began applying theoretical concepts like VLANs for network segmentation and trunking. This lab now serves as my platform to demonstrate practical skills in managing enterprise-grade hardware (HPE, Aruba) and designing a secure network, effectively bridging the gap between theory and real-world application.

Components & Configuration

TP-Link WISP Router

Functions as a bridge for the wifi.id@home connection, forwarding the network to the main router.

HPE MSR930 Router

The main router that handles all network functions (DHCP, DNS, Routing, Firewall, Port Forwarding).

[Planned Upgrade] FortiGate 30E Firewall

Planned for integration as the primary NGFW. It will be responsible for security policies, inter-VLAN traffic inspection, web filtering, and IPS.

Aruba AP11 Access Point

A Multi-VLAN Access Point serving multiple SSIDs for various VLANs (Core, IoT, Guest).

HPE 4210G Switch

The central managed switch responsible for traffic segmentation. It uses a trunk port to deliver all VLANs to the Aruba AP, while access ports are assigned based on device type.

Benefits & Solutions

Multi-Device Connection

Overcomes the limitation of wifi.id@home, which only allows one account per device. With this setup, one account can be used for many devices simultaneously.

Home WiFi Solution

Provides an internet connection solution for homes without dedicated WiFi service. All devices in the house can connect to the internet stably.

Smart Home Ready

This network infrastructure enables a future smart home setup. With a capable router, all IoT devices can be connected and managed easily.

Learning Environment

This setup serves as a real-world laboratory for learning networking, implementing CCNA concepts, and experimenting with various network configurations.

Important Note

The HPE switch is only used as needed due to its significant power consumption. This setup allows for flexibility in managing devices with optimal energy efficiency.
